Accelerep API Reference

Full reference for the Accelerep REST API v1. All endpoints require a valid API key issued from Settings → Integrations → API keys.

Base URL
https://salespipelinetracker.com/.netlify/functions
Auth header
Authorization: Bearer <key>
Content type
application/json
Versioning
URL-stable · breaking changes increment version

Authentication

Every request must include your API key in the Authorization header as a Bearer token. Keys are issued from Settings → Integrations → API keys and require Admin role to create.

curl https://salespipelinetracker.com/.netlify/functions/opportunities \
  -H "Authorization: Bearer spt_live_a1b2c3..." \
  -H "Accept: application/json"
Key format: Keys are prefixed spt_live_ followed by 64 hex characters. Keys are hashed with SHA-256 before storage — Accelerep never stores the plaintext. Each key is shown in full once at creation time only.
Key scope: Keys currently grant read/write access to all resources within the workspace. Scope granularity is on the roadmap. Treat keys as workspace credentials and rotate them if compromised.

Errors & Status Codes

All errors return JSON with an error string. HTTP status codes follow REST conventions.

200
OK — request succeeded. Body contains the resource or collection.
201
Created — resource was successfully inserted. Body contains the created record.
204
No content — used for OPTIONS preflight responses.
400
Bad request — missing or invalid fields. Check the error field for details.
401
Unauthorized — missing, expired, or revoked API key.
403
Forbidden — key is valid but the operation requires a higher-privilege role (e.g. Admin-only endpoint).
404
Not found — the resource does not exist or belongs to a different organization.
405
Method not allowed — HTTP method is not supported on this endpoint.
409
Conflict — e.g. attempting to revoke a key that is already revoked.
500
Internal server error — unexpected failure. Contact support if this persists.
// Error response shape
{
  "error": "id is required"
}

Pagination

Collection endpoints currently return all records for the authenticated organization (up to 500 for audit log). Cursor-based pagination is on the roadmap. For large datasets, use the ?limit parameter where documented.

Rate Limits

Rate limits are enforced per API key and per organization. Default limits: 1,000 requests/minute for standard keys. Exceeding limits returns 429 Too Many Requests with a Retry-After header.

PUT /.netlify/functions/opportunities Update an opportunity

Upserts an opportunity by id. Triggers opportunity.stage_changed, opportunity.won, or opportunity.lost webhooks as appropriate. Sends email notifications to the rep on stage changes.

Upsert behaviour: If a record with this id doesn't exist, it will be created. If it does exist, only the fields included in the request body are updated (other fields are preserved).
curl -X PUT https://salespipelinetracker.com/.netlify/functions/opportunities \
  -H "Authorization: Bearer spt_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "id": "opp_newdeal_001",
    "stage": "Proposal",
    "probability": 60,
    "forecastCategory": "best_case"
  }'
// Response 200
{
  "opportunity": { /* updated opportunity object */ }
}
DELETE /.netlify/functions/opportunities?id=<id> Delete an opportunity

Permanently deletes an opportunity. Pass ?id= for a single record.

curl -X DELETE \
  "https://salespipelinetracker.com/.netlify/functions/opportunities?id=opp_newdeal_001" \
  -H "Authorization: Bearer spt_live_..."
// Response 200
{ "success": true }

Webhooks

Webhooks allow your backend to receive real-time notifications when CRM events occur. Each subscription defines an endpoint URL and the list of event types it subscribes to. Payloads are signed with HMAC-SHA256.

Supported event types

EventFired when
opportunity.createdA new opportunity is created via the API or the app.
opportunity.stage_changedAn opportunity moves to a new pipeline stage.
opportunity.wonStage is set to "Closed Won".
opportunity.lostStage is set to "Closed Lost".
lead.createdA new lead is created.
lead.convertedA lead's status is first set to "Converted".
task.overdueA task passes its due date without being completed.
task.completedA task is marked complete.
spiff.claimedA SPIFF incentive is claimed by a rep.

Payload format

{
  "event":     "opportunity.stage_changed",
  "timestamp": "2025-03-15T14:22:00.000Z",
  "org_id":    "org_xyz",
  "data": {
    // resource-specific fields — see each event type
  }
}

Verifying signatures

Every request includes an X-SPT-Signature header and an X-SPT-Event header. Always verify the signature before processing the payload.

Important: Never process a webhook payload without verifying its signature. Unverified webhooks can be spoofed.
// Node.js — verify signature
const crypto = require('crypto');

function verifyWebhook(rawBody, signature, secret) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret)
    .update(rawBody)  // raw request body string, not parsed JSON
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature), Buffer.from(expected)
  );
}

// Express handler
app.post('/webhooks/accelerep', express.raw({ type: 'application/json' }), (req, res) => {
  const sig = req.headers['x-spt-signature'];
  if (!verifyWebhook(req.body.toString(), sig, process.env.WEBHOOK_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  const event = JSON.parse(req.body);
  console.log('Received:', event.event, event.data);
  res.status(200).send('OK');
});

API Keys

API keys are managed via the api-keys endpoint. All operations require Admin role. Keys are hashed with SHA-256 before storage — the plaintext is returned only at creation time.

GET /.netlify/functions/api-keys List API keys

Returns all keys for the org. keyHash is never returned — only keyPrefix (the first 16 characters) for identification.

// Response 200
{
  "keys": [
    {
      "id": "apikey_1234567890_abc",
      "name": "Production sync",
      "keyPrefix": "spt_live_a1b2c3d",
      "scopes": ["read"],
      "createdBy": "user_abc",
      "createdAt": "2025-01-14T10:00:00.000Z",
      "lastUsedAt": "2025-03-15T14:22:00.000Z",
      "revokedAt": null
    }
  ]
}
POST /.netlify/functions/api-keys Create an API key
Store immediately: The plaintextKey is returned only in this response. It is hashed before storage and cannot be retrieved again.
-d '{ "name": "BI pipeline sync" }'
// Response 201
{
  "key": {
    "id": "apikey_1234_xyz",
    "name": "BI pipeline sync",
    "keyPrefix": "spt_live_a1b2c3d",
    "scopes": ["read"],
    "createdAt": "2025-03-15T14:00:00.000Z"
  },
  "plaintextKey": "spt_live_a1b2c3d4e5f6..."
}
DELETE /.netlify/functions/api-keys?id=<id> Revoke an API key

Soft-revokes the key by setting revokedAt. The key stops working immediately. Revocation is permanent — create a new key if access is needed again.

curl -X DELETE \
  "https://salespipelinetracker.com/.netlify/functions/api-keys?id=apikey_1234_xyz" \
  -H "Authorization: Bearer spt_live_..."
// Response 200
{ "success": true }
